# Visual QA Checklist (Pencil parity)

Use this checklist before accepting frontend changes.

## Global
- [ ] App uses dark premium palette.
- [ ] Primary accent is neon-lime for key actions.
- [ ] Text contrast is readable in cards and surfaces.
- [ ] Border radius and spacing are consistent across screens.
- [ ] Typography hierarchy matches references (title, section, body, meta).

## Home
- [ ] Desktop keeps left sidebar and two-column content feel.
- [ ] Mobile keeps compact summary blocks and clear CTA.
- [ ] Main CTA is visually dominant.

## Training
- [ ] Routines list has normal/loading/empty/error states.
- [ ] Builder shows exercises and sets with clear edit affordances.
- [ ] Session active supports inline set updates.
- [ ] Finish-session confirmation modal appears and is legible.

## Diet
- [ ] Day layout is grouped by meals.
- [ ] Macro/calorie totals are visible.
- [ ] IA estimate state with confidence is represented (when implemented).

## Measurements
- [ ] Weight and circumference entries are readable and scannable.
- [ ] Historical ordering and date emphasis are clear.

## Chat and BYOK
- [ ] Chat enabled state matches visual style.
- [ ] Chat disabled state appears when no BYOK key.
- [ ] BYOK settings clearly show provider status and actions.

## Auth
- [ ] Login/register/recover/verify share same visual language as app shell.
- [ ] Input, helper, and error states are consistent.

## Responsive
- [ ] Desktop and mobile layouts match corresponding design references.
- [ ] No clipped content at common breakpoints.
- [ ] Touch targets are large enough on mobile.
